In order to make it easy to get started, we have sample bots bellow which are already set up and simply need to be uploaded to Heroku or another hosting service. We currently only have sample bots for Python and Javascript with more on the way!

Bot Requierments

When creating your bot, make sure your bot has a single endpoint that should be used when calling your bot. When adding your bot to your profile please be sure to enter the entire URL including the endpoint. For example if your endpoint is http://my-url/bet you would enter that as your URL.

You are free to handle the request from Probot Playground however you would like, however, your bot must respond within five seconds. Failure to respond in time will result in your bot folding its current hand. If your bot errors when called it will forfeit the current hand and be removed from the table.

Bet Response

When called you must return a JSON object with some required and optional parameters.

  • bet - required: This is the amount you would like to bet. If less than 0 will default to folding, and if larger then your current chips will default to going all in.

  • leave - optional: This is an optional parameter indicating if you would like to leave the table. You will still finish the current hand before leaving.

Feel free to look at the example bots for guidance on implementation.

Joining Tables

Currently, you may join tables through the lobby page once logged into your account. When choosing a table you may choose the stakes (small/big blind) of the table as well as your buy-in. Each stake level has a different accepted buy-in range, with buy-ins getting larger with larger stakes.

Game Data

The data your bot receives will be a JSON object consisting of a ‘state’ and ‘history’ attribute. The ‘state’ is a JSON object of the current state of the game. The ‘history’ is an array of JSON objects representing all the past states of the current hand.
